# Iron Rules（絶対ルール）

これらのルールは例外なく常に遵守する。違反はコミットをブロックする。

## Must Always

1. **日本語は直接文字で書く。** Unicode escape (`\uXXXX`) は一切使用禁止。
2. **1機能1ブランチ。** 複数の無関係な変更を1ブランチに混ぜない。
3. **PRは手動。** Claude Codeが勝手にPRを作成・マージしない。
4. **既存パターンに従う。** 新しいパターンを発明する前に、既存コードベースの慣習を確認する。
5. **仕様書に書かれた変更だけを行う。** 指示されていない改善・最適化・リファクタリングは禁止。
6. **変更を加える前に影響範囲を確認する。** importされている箇所、データスキーマ、ユーザーの指示を調査してから編集する。

## Must Never

1. **未要求のリファクタリングを行わない。** 「ついでに整理」は禁止。
2. **console.logを残さない。** デバッグ用のログは必ず除去（意図的なエラーハンドリングの `console.error` には `// keep` コメントを付ける）。
3. **APIキー・シークレットをソースコードに含めない。** 環境変数を使用する。
4. **設定ファイルを勝手に変更しない。** eslint設定、prettier設定、vite設定、package.jsonの変更は太一さんの承認が必要。
5. **テストされていない変更をpushしない。** 少なくともビルドが通ることを確認する。
6. **画像参照やClaude.aiの会話コンテキストを仕様書に含めない。** Claude Code向け指示書はテキスト完結。
