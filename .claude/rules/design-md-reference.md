# DESIGN.md 参照ルール

UI の新規実装・改修・レビュー時は `/DESIGN.md` を **最初に参照** する。

## 必須参照タイミング

以下の作業時には必ず DESIGN.md を参照する:

- 新規 UI コンポーネントの実装
- 既存 UI コンポーネントのスタイル変更
- CSS 変数・トークン値の追加・変更
- 画面レイアウトの変更
- 色・タイポ・余白・角丸・影・モーションの指定
- 画面別の質感・雰囲気の実装

## 参照順序

1. `/DESIGN.md` §1「Overview & Role Separation」で役割分離を確認
2. 該当する章（§2 Color, §3 Typography, §4 Spacing, §5 Shadow, §6 Radii, §7 Components, §8 Motion, §9 Screens, §10 Don'ts）を参照
3. 必要に応じて `/DESIGN_PHILOSOPHY.md`（哲学）と `.claude/skills/molkky-ui-design/SKILL.md`（実装ガイドライン）を追加参照

## トークン使用の必須ルール

- **生値の直書き禁止**: `#xxxxxx`, `rgb(...)`, `12px`, `0.25s` 等の生値を CSS・JSX に直書きしてはならない
- **DESIGN.md 定義のトークン使用**: `var(--neutral-0)`, `var(--text-base)`, `var(--space-4)` 等の CSS 変数を使用
- **新規トークンが必要な場合**: 先に DESIGN.md に追加してから実装する（手順の逆転禁止）

## 改善提案時の出典明記（§1.5）

DESIGN.md を参照して改善案・実装指示書・レビュー結果を出力する際、必ず根拠の出典を明記する:

- DESIGN.md の章・節番号（例: 「§9.4.9 スコア分布ヒートマップに基づき」）
- DESIGN_PHILOSOPHY.md の哲学
- 現状コードの Audit（DESIGN_AUDIT.md）
- 外部デザイン参照（具体的に何を参照したか）
- 過去の議論・決定

詳細は DESIGN.md §1.5「改善提案時の出典明記ルール」を参照。

## §10 Don'ts の禁止事項

以下は絶対に違反してはならない:

- プレイヤー色を識別以外の用途に使う（§10.2）
- Wind Ramp 色をスコアヒートマップに流用（§10.2）
- ダーク化対象をハイブリッド方針から逸脱させる（§10.2）
- パルス/脈動を 1 画面 2 箇所以上に同時適用（§10.2）
- GameScreen ヘッダーの Wind Vector Widget を他画面へ転用（§10.2）
- ヘキサゴン配列スキットルボタンの改変（§10.2）
- 11・12 点以外の棒にゴールド強調を適用（§10.2）
- 直射日光視認性を絶対条件として扱う（§10.2）
- レーダーチャートのモルックキャラ背景を削除（§10.2）
- `--font-instrument` の無差別使用（§10.2）
- Tier 境界の崩壊（§10.2）
- クラウド同期エリアの特別演出を他カードに転用（§10.2）

その他、§10.1「AI 生成UI回避チェックリスト」（原則1〜5）を常に意識する。

## Step 4 第2弾以降の視覚変化保留項目

以下は Step 4 第1弾時点では実コードの定義値変更が **保留** されている。該当変数を新規実装で使う場合、現状値と理想値の差分を認識した上で実装すること（DESIGN.md §2.10.3 参照）:

- `--accent-red`: 現状 `#d93a5e`、理想値 `#e74c3c`
- `--radius-sm`: 現状 `6px`、理想値 `8px`
- `--radius-lg`: 現状 `14px`、理想値 `12px`
- `--radius-xl`: 現状 `18px`、理想値 `16px`
